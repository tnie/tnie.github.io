devcpp 5.1.1 在 win8.1 x64 系统上

发表于 2017-04-18 更新于 2023-11-28 分类于 tools

  1. 编译器 TDM-GCC 4.9.2 64-bit Release/Debug，可能对 64 位处理的不是很好，经常报 gdb.exe 崩溃。因此建议不使用用 64 位的编译配置，包括 Release、Debug；

  2. 使用 TDM-GCC 4.9.2 32-bit Release/Debug，报错缺失 zlib1.dll 文件，这一点可能是 32 位编译器对 64 位系统的兼容性有限，参考 计算机中丢失 ZLIB1.DLL 去网上下载 dll 文件拷贝到相应目录即可。然后就可以正常使用 devcpp 了。

  3. 打开 devcpp，将编译器选择 TDM-GCC 4.9.2 32-bit Debug ，每次重启之后报错：

    解决方法：在 mingw64 创建一个 lib32 文件夹就可以了。
